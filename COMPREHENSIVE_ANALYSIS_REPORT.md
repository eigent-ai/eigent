## 核心发现

### 1. 成功率与步数的权衡

```
模型                 成功率    平均步数    效率指数(成功率/步数)
claude-opus-4-5     80.00%    33.54      2.38  🥇
gemini-3-pro-preview 73.33%   20.36      3.60  🥈
gpt-5.1             66.67%    16.33      4.08  🥉
```

**注**: 效率指数越低，说明用更多步数换取了更高成功率

### 2. 模型策略差异

| 模型 | 策略类型 | 特点 |
|------|---------|------|
| **claude-opus-4-5** | 稳健型 | 使用最多步数，确保任务完成 |
| **gemini-3-pro-preview** | 平衡型 | 在速度和准确性间取得平衡 |
| **gpt-5.1** | 激进型 | 追求效率，但牺牲部分成功率 |

### 3. 子任务成功率分析

所有模型的子任务成功率都显著高于整体任务成功率，这表明：
- 大多数任务能完成部分子任务
- 失败往往发生在关键步骤
- 70-80%的子任务完成率是常见水平

### 4. 步数分布

```

gemini-3-pro-preview: ████████████████████                  20.36
gpt-5.1:             ███████████████                        16.33
```

### 5. 数据覆盖差异

⚠️ **注意**: Browser log数据与评估数据的任务数存在差异：
- claude-opus-4-5: 13个 browser log vs 15个评估任务
- gemini-3-pro-preview: 14个 browser log vs 15个评估任务

这可能是因为：
- 部分任务的browser log未被提取

- gpt-5.1有额外的测试任务
